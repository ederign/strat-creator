# Human Review Guide

> **Audience**: Staff engineers and architects reviewing strategies after CI processing.
>
> | Label | Meaning |
> |-------|---------|
> | `strat-creator-rubric-pass` | CI approved — ready for human sign-off |
> | `strat-creator-needs-attention` | CI flagged — needs fixes before sign-off |
> | `strat-creator-human-sign-off` | Human signed off — feature-ready |

## Quickstart

```bash
# Rubric-pass (approved by CI) — review and sign off
claude "/strategy-pull RHAISTRAT-1520"        # Pull into local/
claude "/strategy-review"                      # Optional: re-score locally
claude "/strategy-signoff RHAISTRAT-1520"      # Sign off → feature-ready

# Needs-attention (flagged by CI) — fix, resubmit, then sign off
claude "/strategy-pull RHAISTRAT-1520"        # Pull into local/
# ... edit Staff Engineer Input or add an overlay ...
claude "/strategy-refine"                      # Re-generate strategy
claude "/strategy-review"                      # Re-score
claude "/strategy-push RHAISTRAT-1520"         # Resubmit to CI
# Wait for CI to approve, then:
claude "/strategy-signoff RHAISTRAT-1520"      # Sign off → feature-ready
```

## Scoring

CI scores every strategy on 4 dimensions (Feasibility, Testability, Scope, Architecture), each 0–2.

| Verdict | Trigger | Meaning |
|---------|---------|---------|
| **APPROVE** | total >= 6, no zeros | Ready for human sign-off |
| **REVISE** | total >= 3, at most 1 zero | Fixable — a dimension or two scored low |
| **REJECT** | total < 3 or 2+ zeros | Fundamental problems |

## Setup

1. Install [Claude Code](https://docs.anthropic.com/en/docs/claude-code)
2. Set Jira credentials:
   ```bash
   export JIRA_SERVER="https://redhat.atlassian.net"
   export JIRA_USER="your-email@redhat.com"
   export JIRA_TOKEN="your-atlassian-api-token"  # https://id.atlassian.com/manage-profile/security/api-tokens
   ```
3. Clone the repo: `git clone https://github.com/ederign/strat-creator.git && cd strat-creator`

## Workflow

### 1. Pull the strategy

```bash
claude "/strategy-pull RHAISTRAT-1520"
```

This fetches into `local/` (gitignored, isolated from CI):
- `local/strat-tasks/RHAISTRAT-1520.md` — strategy with frontmatter
- `local/strat-originals/RHAIRFE-NNNN.md` — linked RFE source
- `local/strat-reviews/RHAISTRAT-1520-review.md` — CI review (if any)

### 2. Read the review

Open `local/strat-reviews/RHAISTRAT-NNNN-review.md` or browse the [dashboard](https://strat-dashboard-0f1209.gitlab.io/). Check:
- **Score table** — which dimensions failed?
- **Prose reviews** — what did each reviewer flag?

### 3. Fix inputs (needs-attention only)

**Never edit strategy text directly.** Edit the inputs, rerun the pipeline, and let it regenerate the output.

| Fix Path | When | What to Edit |
|----------|------|-------------|
| **Overlay** (recommended) | Wrong dependencies, outdated versions, component gaps affecting multiple strategies | Create an overlay in `opendatahub-io/architecture-context`. See the [Overlays README](https://github.com/opendatahub-io/architecture-context/blob/main/overlays/README.md) for format, testing, and submission. |
| **Staff Engineer Input** | Issue specific to this strategy (effort estimate, scope, test criteria) | `## Staff Engineer Input` section in `local/strat-tasks/RHAISTRAT-NNNN.md` |
| **Architecture context** | Major structural changes (new components, boundary changes) | Contact maintainers (James Tanner, Kevin Bader, Luca Burgazzoli) |

**Prefer overlays.** They fix the root cause for all future strategies, not just the one you're reviewing. Use Staff Engineer Input only for one-off fixes.

Example Staff Engineer Input:

```markdown
## Staff Engineer Input

Effort estimate is too low. This touches the inference controller
and the model registry, two teams need to coordinate. Estimate
should be M not S. Add a dependency on RHAISTRAT-1200.
```

### 4. Rerun locally

The skills auto-detect `local/` files — no flags needed, no Jira writes:

```bash
claude "/strategy-refine"
claude "/strategy-review"
```

Check `local/strat-reviews/` for updated scores. Repeat from step 3 if needed.

### 5. Push or sign off

| CI Verdict | Command | What Happens |
|------------|---------|-------------|
| `needs-attention` | `/strategy-push RHAISTRAT-NNNN` | Pushes fixes to Jira, removes `needs-attention`. CI re-evaluates. Come back to sign off after CI approves. |
| `rubric-pass` | `/strategy-signoff RHAISTRAT-NNNN` | Pushes any edits, adds `strat-creator-human-sign-off`. Strategy is feature-ready. |

## Key Rules

1. **Edit inputs, not outputs.** The strategy text is regenerated by refine — direct edits get overwritten.
2. **Prefer overlays over one-off fixes.** They're durable and benefit all future strategies.
3. **Always rerun refine before review.** Review scores what refine produces, not what you wrote.
4. **Needs-attention must go back through CI.** Use `/strategy-push`, then `/strategy-signoff` after CI approves.
5. **Sign-off requires CI approval.** `strat-creator-human-sign-off` requires `strat-creator-rubric-pass`.
